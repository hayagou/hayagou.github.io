Table and Form

Posted by hayagou on 2011-01-02 · 2 mins read

[부트스트랩] 테이블과 폼 태그 스타일

tr 	//table row, 가로 
td 	//table data, 세로
th 	//table header

그리드 레이아웃의 row 를 그대로 적용해서 네이밍 한것 “td” 가 table col이 아니고 table data인 이유는 데이터가 들어가는 부분이라서

Table Class들

<table class="table">
  ...
</table>

table.table : 약간의 패딩과 수평 구분선

table.table-striped : 줄무늬 행

table.table-striped : 선있는 테이블

table.table-hover : 행 hover, 마우스올리면 불들어옴

table.table-condensed : 좁은폭 테이블


Table Row에 효과를 넣고 싶을

클래스 설명
.active 특정한 행이나 셀에 hover 했을 때와 같은 색을 입힙니다.
.success 성공적이거나 긍정적인 액션을 의미합니다.
.info 중립적인 정보의 변화나 액션을 의미합니다.
.warning 주의가 필요한 경고를 의미합니다.
.danger 위험하거나 부정적일수 있는 액션을 의미합니다. 
<!-- On rows -->
<tr class="active">...</tr>
<tr class="success">...</tr>
<tr class="warning">...</tr>
<tr class="danger">...</tr>
<tr class="info">...</tr>

<!-- On cells (`td` or `th`) -->
<tr>
  <td class="active">...</td>
  <td class="success">...</td>
  <td class="warning">...</td>
  <td class="danger">...</td>
  <td class="info">...</td>
</tr>


폼태그 스타일

form.form-inline
	div.form-group.has-success.has-feedback
		div.input-group
			span.input-group-addon
				span.glyphicon glyphicon-*(들어가 이미지 명)
			input.form-control

form-inline : 폼의 요소들을 한줄로 정 form-horizontal : 폼의 요소들을 수평으로 정렬 form-group : form태그 안의 최적의 여백을 위해 input-control : 조작하고싶은 요소 control-label : 레이블테그가 form-group이 가지고있는 스타일을 따라 has-스타일이름 : 입력창에 스타일을 줌 input-group : 인풋 테그 스타일을 줌 input-group-addon : 인풋 태그에 보조아이콘 추가
