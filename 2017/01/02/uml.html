UML과 클래스 다이어그램

Posted by hayagou on 2017-01-02 · 2 mins read

[디자인패턴] UML

UML의 구성 요소

  • 사물(Things) : 기본 요소
  • 관계(Relationship) : 사물 간의 관계
  • 다이어그램(Diagram) : 사물관 관계를 도형으로 표현


사물(Things)

사물은 추상적 개념으로서 모델에서 가장 중요

  • 구조사물 : 시스템의 구조를 표현하는 사물
  • 해동사물 : 시스템의 행위를 표현하는 사물
  • 그룹사물 : 개념을 그룹화하는 사물
  • 주해사물 : 부가적으로 개념을 설명하는 사물

관계

요소가 서로 연관되는 방법

관계 표시 설명
연관관계(association) 두 사물간의 의미적 관계, 하나의 요소의 변화는 다른 하나에 영향을 미치는 두가지 사이의 관계
일반화 관계(generalization) 클래스와 인터페이스 사이에서 상속관계를 설명
집약관계(aggregation) 라이프 타임은 독립적전체, 객체가 없어져도 부분 객체는 없어지지않는다
합성 관계(composition) 라이프 타임은 의존적, 전체 객체가 없어지면 부분객체도 없어진다.
의존관계(dependency) 연관 관계와 같음, 차이점은 두 클래스의 관계가 한 메서드를 실행하는 동안과 같은, 매우 짧은 시간만 유지된다는 점이다.
실체화 관계(realization) 인터페이스와 클래스 컴포넌트 사이의 관계를 지정
