State Pattern

Posted by hayagou on 2017-01-13 · 1 min read

[디자인패턴] State Pattern


의도

객체의 내부 상태에 따라 스스로 행동을 변경할 수 있게끔 허가하는 패턴으로, 이렇게 하면 객체는 마치 자신의 클래스를 바꾸는 것처럼 보임

  • 유한상태객체(FSM, Context)로 부터 상태를 분리하고 행동을 상태에게 위임함

활용성

  • 객체의 행동이 상태에 따라 달라지고, 런타임에 행동이 바뀔때

  • 객체의 상태에 따라 달라지는 다중 분기 조건 처리가 맣이 들어 있을 때

(cf) Strategy Pattern - class diagram이 동일함

[전략 패턴] client가 context의 행동(strategy)을 설정(변경)함, (용도) context의 유연한 행동 변경

[상태 패턴] context 자신이 상태 변경을 통해 행동을 변경함, (용도) (context의) case analysis의 대안

State Pattern 참고

Finite State Machine(FSM)

Basic concepts

  • 한 순간 오직 하나의 상태를 가짐
  • 한 상태에서 다른 상태로 전이 될 수 있음
  • 상태전이는 외부입력(사건,행동)에 의해 야기

Mealy macine -> 6-tuple(상태집합, 초기상태, 유한입력집합, 유한출력집합, 전이함수, 출력함수)

Moore machine -> 출력함수: 상태집합 -> 출력집합(출력은 오직 현재 상태에만 의존함)

사건 event - 시스테멩 영향을 주는 것(타입)

상태 변수 state variable - 상태를 나타내느 변수, 두 종류가 있음

  • Enumeratable state variable) 질적qualitative 상태 ex> 열거형

  • Extended state variable) 양적quantitative 상태 ex>int

  • Problem) 상태변수의 변경이 항상 state transition을 야기한다면 state explosion
  • Solution) 가드 조건 - 확장된 상태변수와 사건 파라미터로 이루어진 boolean expression

가드는 확장된 상태변수의 state explosion을 방지할 수 있다

  • 상태 전이 state transition : 상태 x 행동 x 가드 -> 상태
