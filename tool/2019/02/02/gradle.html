IntelliJ에서 Gradle로 자바 프로젝트 만들기

Posted by hayagou on 2019-02-02 · 2 mins read

Gradle Build Tool

Gradle Wrapper란?

  • 프로젝트를 새로운 환경에서 개발할때 라이브러리들을 옮길 필요없이 바로 빌드를 가능하게 해줌 스프링 Maven프로젝트의 pox.xml에 기술하면 라이브러리가 추가되는거랑 똑같다.


IntelliJ로 Java 프로젝트 만들기 with Gradle

인텔리제이를 통해 gradle 프로젝트를 만들고 간단하게 추가하는 방법이다.. 구글에 검색하면 다른 방법들이 많은데 간단하게 추가한 방법을 기록했다.

  1. 인텔리제이를 키고,

  2. gradle로 프로젝트를 만든다.

  3. 프로젝트 설정들을 하고,

  4. 조금 기다리면 프로젝트가 생성된다.

  5. 실행될 메 클래스를 만들고, Project Name > src > build.gradle 로 가서 원하 의존성을 추가해준다.

  6. 오른쪽의 새로고침 버튼을 누르고 외부 라이브러리가 추가 된다.


의존 라이브러리

저장소에서 필요한 라이브러리를 적어주고 사용할수 있는 것이 dependencies라는 문이다. 기본 형태는

dependencies{
	.... 라이브러리 ....
}

이런 형식이고 {} 안에 종속 라이브러리에 대한 기술을 한다.


컴파일시 의존 라이브러리 문법

compile 'org.json:json:20190722'

compile~ 컴파일시에 사용하는 라이브러리를 지정한다.


테스트 컴파일시 의존 라이브러리 문법

testCompile 'junit:junit:4.12'

testCompile~ 테스트 컴파일시에 사용하는 라이브러리를 지정한다. (단위 테스트할때 사용)

라이브러리 지정하는 방법

위 사진 처럼 작성한다. 라이브러리 지정하는 방법에 규칙이 있다.

'그룹 : 이름 : 버젼 '

이런 형태로 기술한다.

쓰고 싶은 라이브러리는 메이븐레파지토리에서 검색을 하여 추가한다.

에를 들어서 Json 라이브러리를 추가하여 사용하고 싶다면,

  1. 메이븐 레파지토리에서 json을 검색후 그륩, 이름, 버젼을 찾아낸다.

  2. build.gradle 을 열고 dependencies에 추가한다.

compile       'org.jason :      json :            20190722'
                    ↓              ↓                  ↓
compile gruop:'org.jason',name:'jason',version:'20190722'

이러한 형식으로 사용한다.
